# 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品定位
本地私有化部署的Docker镜像管理系统，参考Docker Registry和Harbor项目设计，提供轻量级的镜像仓库管理能力。

### 1.2 目标用户
- 开发团队：需要私有化镜像存储和管理的开发团队
- 运维人员：负责镜像部署和维护的运维人员
- 企业管理：需要集中管理Docker镜像的企业用户

### 1.3 核心价值
- 简化Docker镜像的本地化存储和管理
- 提供基础的权限控制和访问管理
- 支持单体应用部署，降低运维复杂度
- 提供镜像版本管理和标签管理功能

## 2. 功能需求

### 2.1 仓库管理
**优先级：P0**

| 功能项 | 描述 |
|--------|------|
| 创建仓库 | 支持创建新的镜像仓库，仓库名称必须为英文 |
| 编辑仓库 | 支持修改仓库信息（默认library仓库除外） |
| 删除仓库 | 支持删除仓库及其下所有项目（默认library仓库除外） |
| 默认仓库 | 系统初始化时创建library作为默认仓库，不可编辑和删除 |

**数据模型：**
```
仓库（Repository）
├── 仓库名称（唯一，英文）
├── 创建时间
├── 更新时间
└── 项目列表
```

### 2.2 项目管理
**优先级：P0**

| 功能项 | 描述 |
|--------|------|
| 创建项目 | 在指定仓库下创建项目，项目名称必须为英文 |
| 编辑项目 | 修改项目信息，包括公开/私有属性 |
| 删除项目 | 删除项目及其下所有镜像 |
| 公开/私有设置 | 设置项目访问权限，公开项目允许访客拉取 |

**数据模型：**
```
项目（Project）
├── 项目名称（仓库内唯一，英文）
├── 所属仓库
├── 访问级别（公开/私有）
├── 创建时间
├── 更新时间
└── 镜像列表
```

### 2.3 镜像管理
**优先级：P0**

| 功能项 | 描述 |
|--------|------|
| 镜像推送 | 支持通过Docker命令推送镜像到指定仓库/项目 |
| 镜像拉取 | 支持通过Docker命令从仓库拉取镜像 |
| 修改标签 | 支持重新修改镜像标签 |
| 添加描述 | 支持为镜像添加描述信息 |
| 删除镜像 | 支持删除指定镜像及其所有标签 |

**镜像路径格式：**
```
域名或IP/仓库名称/项目名称:镜像标签
示例：192.168.1.100/library/mysql:5.7
```

**数据模型：**
```
镜像（Image）
├── 镜像名称
├── 所属项目
├── 标签列表
│   ├── 标签名称
│   ├── 镜像摘要（digest）
│   ├── 大小
│   ├── 推送时间
│   └── 推送用户
├── 描述信息
├── 创建时间
└── 更新时间
```

### 2.4 用户管理
**优先级：P1**

| 功能项 | 描述 |
|--------|------|
| 初始化管理员 | 系统初始化时创建admin管理员账户 |
| 首次登录改密 | 管理员首次登录时强制修改初始密码 |
| 用户增删改查 | 管理员可管理普通用户账户 |
| 分配角色 | 管理员可为用户分配角色 |

### 2.5 角色管理
**优先级：P1**

| 功能项 | 描述 |
|--------|------|
| 初始化角色 | 系统预置管理员、开发者、访客三种角色 |
| 创建角色 | 支持创建自定义角色 |
| 编辑角色 | 支持修改角色信息和权限 |
| 删除角色 | 支持删除自定义角色（预置角色除外） |

**预置角色：**
| 角色 | 描述 |
|------|------|
| 管理员 | 拥有系统所有权限 |
| 开发者 | 可推送和拉取镜像，修改标签，删除镜像 |
| 访客 | 仅可拉取公开镜像 |

### 2.6 权限管理
**优先级：P1**

| 权限项 | 管理员 | 开发者 | 访客 |
|--------|--------|--------|------|
| 用户管理 | ✓ | ✗ | ✗ |
| 角色管理 | ✓ | ✗ | ✗ |
| 权限管理 | ✓ | ✗ | ✗ |
| 仓库管理 | ✓ | ✗ | ✗ |
| 项目管理 | ✓ | ✗ | ✗ |
| 镜像管理 | ✓ | ✓ | ✗ |
| 系统设置 | ✓ | ✗ | ✗ |
| 日志审计 | ✓ | ✗ | ✗ |
| 推送镜像 | ✓ | ✓ | ✗ |
| 拉取镜像 | ✓ | ✓ | ✓（仅公开） |
| 修改标签 | ✓ | ✓ | ✗ |
| 删除镜像 | ✓ | ✓ | ✗ |

## 3. 非功能需求

### 3.1 性能需求
- 镜像推送/拉取速度应接近原生Docker Registry
- 页面加载时间 < 2秒
- 支持并发用户数 >= 50

### 3.2 安全需求
- 用户密码加密存储
- 支持HTTPS访问
- 操作日志记录
- 会话超时管理

### 3.3 可靠性需求
- 系统可用性 >= 99%
- 数据备份支持
- 异常恢复能力

### 3.4 兼容性需求
- 兼容Docker CLI 18.09+
- 支持主流浏览器（Chrome、Firefox、Edge）

## 4. 数据架构

### 4.1 层级结构
```
仓库（Repository）
└── 项目（Project）
    └── 镜像（Image）
        └── 标签（Tag）
```

### 4.2 示例结构
```
library（仓库）
├── mysql（项目）
│   ├── mysql:5.7
│   └── mysql:8.0
├── redis（项目）
│   ├── redis:6.0
│   └── redis:7.0
└── nginx（项目）
    ├── nginx:1.20
    └── nginx:1.21
```

## 5. 技术约束

| 组件 | 技术选型 |
|------|----------|
| 后端框架 | SpringBoot + Java |
| 前端框架 | React + TypeScript |
| 数据库 | SQLite3 |
| 缓存 | Memcached |
| 存储 | 本地文件系统（FS） |

## 8. 验收标准

### 8.1 功能验收
- 仓库、项目、镜像的增删改查功能正常
- 镜像推送和拉取功能正常
- 用户权限控制正确

### 8.2 性能验收
- 镜像推送速度 >= 10MB/s
- 镜像拉取速度 >= 20MB/s
- 页面响应时间 < 2秒

### 8.3 安全验收
- 未授权访问被正确拦截
- 密码加密存储
- 操作日志完整记录
